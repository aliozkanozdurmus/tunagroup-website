# Contributing

Thank you for improving this repository.

## Workflow

1. Create a feature branch from `main`.
2. Keep commits small and descriptive.
3. Open a pull request with context and validation notes.

## Pull Request Checklist

- [ ] Scope is clear and documented.
- [ ] No secrets or credentials were committed.
- [ ] `.env.example` is updated if configuration changed.
- [ ] Basic lint/test checks were run.
